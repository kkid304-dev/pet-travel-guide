# pet-travel-guide
## 📊 데이터 분석 메모 (개발 중 발견)

한국관광공사 반려동물 동반여행 API(KorPetTourService2)를 직접 탐색하며 확인한 내용.

### 데이터 규모와 분포 (서울, lDongRegnCd=11 기준)
- 동반 가능 장소 전체: **3,179곳**
- 카테고리 분포: **쇼핑+음식점 약 3,100곳(97%)**, 관광지 **56곳(1.8%)**
- → 필터 없는 목록은 사실상 상업시설 리스트가 됨. **카테고리 필터는 필수 기능**
- 최신순(arrange=C) 정렬 시 백화점 입점 매장 일괄 등록 때문에 쇼핑 쏠림 심화

### 동반 조건 데이터의 특성
- 조건(acmpyPsblCpam)은 코드값이 아닌 **자유 텍스트**
  - 예: "전 견종 출입 가능 (맹견의 경우, 입마개 착용 필수)"
- **빈 값인 장소가 다수** → acmpyTypeCd(동반구분)로 대체하는 fallback 구현
- kg 수치가 명시된 장소는 드묾 (서울 50곳 표본에서 0곳)
  - → 무게 파싱만으로는 판정이 거의 갈리지 않음. 단어 기반 해석 병행 필요
- 판정 분포 (50곳 표본): ✅ 48 / ⚠️ 2 / ❌ 0
  - 애초에 "동반 가능한 곳만 모은 DB"라 ✅ 편중은 구조적

### ⚠️ 문장 수집 (judge 고도화 대상)
실데이터에서 발견한, 해석이 필요한 조건 문장들:
- "일부구역 동반가능"
- "맹견 및 대형견 제외 동반 가능"
- "보조견 동반 입장 가능. 야외 좌석과 내부 1층만 허용됨."
- "전 견종 출입 가능 (맹견의 경우, 입마개 착용 필수)"

### 서비스 시사점
1. 문제 정의 검증: 조건이 자유 텍스트 + 결측 다수 → "조건 해석·보완 서비스"의 필요성이 데이터로 확인됨 (지정과제 6번 취지와 일치)
2. 판정 로직 방향: kg 파싱(v2 구현 완료) + 크기/견종 단어 해석(v3 예정)
3. 필수 기능: 카테고리 필터, 정보 미확인 시 "방문 전 문의 권장" 안내
